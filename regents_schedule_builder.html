How each output cell is determined

For each teacher × date × time-slot, the tool evaluates in this order:

  1. Blank check (slots A–D only): if shift is blank → output blank.
  2. Whole-day passthrough: if shift is S, E, R, AB, N/A, or NA → output that code.
  3. Room lookup: find the teacher in panel 2 for this date and slot. If found → output the room number.
  4. Fallback rotation: output H (hallway), L (lunch), or blank based on the shift code, slot, and previous slot.

Fallback rotation rules (when teacher isn't in a room)

SlotRule
AAA → H; others → blank (not on shift yet)
BCC → blank; AA after a room → L; arriving (prev blank) → H
CAA after L → H; BB after H → L; CC → H; else → L
DAA → H; BB just out of a room → H; CC → L; else → H
EAA → off (blank); BB → H if still active; else → H
FAA and BB → off (blank); CC → H

Other things to know

  • Name matching is case-insensitive. "SMITH J", "Smith J", and "smith j" all match.
  • Leave room cells blank in panel 2 if no one is proctoring that slot in that room.
  • Auto-saves to this browser ~½ second after each edit. Close the tab and reopen — your data is still there. Use Export Data for a portable JSON backup.
  • Preserves original formula behavior for valid shifts (AA/BB/CC/S). Improvements: cells that used to return FALSE now return blank; N/A, AB, E, R now produce sensible whole-day output (the original formulas would give random H/L values for these).
  • To use with Excel: Ctrl+C from Excel → Ctrl+V into a textarea here. After Generate, click Copy Output as TSV → Ctrl+V into Excel.
